## 交叉审查清单

### 事实核查

**第一步：提取**（用 Grep 搜出文档中所有含数量词的声明）
- [ ] Grep `\d+\s*(个|张|条|种|层|步|次|份|项|类|行|个方法|个文件|个端点|张表)` 在 docs_project_portrait/ 下搜索
- [ ] 整理成清单（文档 | 行号 | 声明 | 声称值）

**第二步：逐条验证**（每行用 Glob 或 Grep 在源码中数实际值）
- [ ] 表数量 → Glob models + 确认每文件的 Model 类数
- [ ] 枚举数量 → Grep `class.*Enum`
- [ ] 方法数量 → Grep `def ` 统计
- [ ] 端点数量 → Grep `@router\.(get|post|put|delete|patch)`
- [ ] 文件/模块数 → Glob 目录 + 确认
- [ ] 表名/字段名/函数名/类名 → Grep 确认存在
- [ ] 配置默认值 → Read config 确认
- [ ] 执行步骤顺序 → Read 逐行对照

**第三步：修正**
- [ ] 所有不一致的声明已修正
- [ ] 03（事实基准）修正后，其他文档的引用数字已同步
